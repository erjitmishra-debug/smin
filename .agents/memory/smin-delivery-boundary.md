---
name: SMIN delivery boundary
description: The product boundary and truthfulness rule for the mining application’s phased delivery.
---

Phase 1 should only claim functionality for project setup, spatial metadata, drillhole collar intake, validation visibility, local persistence, processing queue state, audit history, sample data, and basic workspace presentation. Geological modelling, estimation, planning, scheduling, economics, reporting, AI, and native packaging require later phases.

**Why:** The source brief explicitly requires module-by-module delivery and prohibits fake calculations or placeholder modules presented as complete.

**How to apply:** Keep later modules visibly marked as pending until their inputs, processing logic, saved outputs, version history, audit records, and tests exist.
